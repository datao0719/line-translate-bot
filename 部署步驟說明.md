# LINE 翻譯機器人 — 部署步驟說明

## 架構概覽

```
LINE App → LINE Platform → Webhook → Render/Railway (Flask) → OpenAI GPT
```

---

## 第一步：申請 LINE Developers 帳號與建立 Channel

1. 前往 [LINE Developers Console](https://developers.line.biz/console/)，用 LINE 帳號登入
2. 點選 **Create a new provider**，輸入名稱（例如：MyTranslateBot）
3. 點選 **Create a new channel** → 選擇 **Messaging API**
4. 填入以下資訊：
   - Channel name：翻譯機器人（或自訂）
   - Channel description：簡短描述
   - Category / Subcategory：自選
5. 建立完成後，進入 Channel 設定頁面

### 取得金鑰

在 Channel 頁面找到：

| 金鑰 | 位置 |
|------|------|
| **Channel Secret** | Basic settings 頁籤 |
| **Channel Access Token** | Messaging API 頁籤 → 最下方 Issue |

> ⚠️ Access Token 需手動點 Issue 按鈕才會產生

---

## 第二步：取得 OpenAI API Key

1. 前往 [platform.openai.com](https://platform.openai.com/)，登入後點選右上角頭像 → **API keys**
2. 點 **Create new secret key**，複製金鑰（只顯示一次，請立即儲存）
3. 確認帳號有足夠的 Credits（新帳號通常有免費額度）

---

## 第三步：部署到 Render（免費）

### 3-1. 上傳程式碼到 GitHub

1. 在 GitHub 建立一個新的 repository（例如 `line-translate-bot`）
2. 將 `line-translation-bot/` 資料夾內的所有檔案推送上去：
   ```bash
   cd line-translation-bot
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/你的帳號/line-translate-bot.git
   git push -u origin main
   ```

### 3-2. 在 Render 建立 Web Service

1. 前往 [render.com](https://render.com/)，用 GitHub 帳號登入
2. 點選 **New** → **Web Service**
3. 連結你的 GitHub repository
4. 設定如下：

   | 欄位 | 值 |
   |------|----|
   | Environment | Python |
   | Build Command | `pip install -r requirements.txt` |
   | Start Command | `gunicorn app:app` |

5. 點選 **Environment Variables**，新增以下三個變數：

   | Key | Value |
   |-----|-------|
   | `LINE_CHANNEL_SECRET` | 步驟一取得的 Channel Secret |
   | `LINE_CHANNEL_ACCESS_TOKEN` | 步驟一取得的 Access Token |
   | `OPENAI_API_KEY` | 步驟二取得的 OpenAI Key |

6. 點 **Create Web Service**，等待部署完成
7. 部署完成後，取得你的服務網址，格式為：`https://你的服務名稱.onrender.com`

---

## 第四步：設定 Webhook

1. 回到 LINE Developers Console → 你的 Channel → **Messaging API** 頁籤
2. 找到 **Webhook URL**，填入：
   ```
   https://你的服務名稱.onrender.com/callback
   ```
3. 開啟 **Use webhook** 開關
4. 點 **Verify** — 應該看到 Success

### 建議關閉的設定

在 Messaging API 頁籤：
- **Auto-reply messages** → 關閉
- **Greeting messages** → 關閉（避免跟 Bot 回覆衝突）

---

## 第五步：加入機器人並測試

1. 在 Messaging API 頁籤找到 **Bot's QR code**，用 LINE 掃描加入好友
2. 傳送訊息測試：
   - 傳送中文 → 應收到英文翻譯 + 關鍵單字介紹
   - 傳送英文 → 應收到繁體中文翻譯 + 語法檢查 + 相似句型

---

## 部署到 Railway（替代方案）

1. 前往 [railway.app](https://railway.app/)，用 GitHub 登入
2. 點 **New Project** → **Deploy from GitHub repo**
3. 選擇你的 repository
4. 在 **Variables** 頁面新增三個環境變數（同 Render 步驟）
5. Railway 會自動偵測 Procfile 並部署

---

## 常見問題

**Q: Webhook Verify 失敗？**  
A: 確認 Render 服務已成功啟動（訪問根網址應看到 "LINE Translation Bot is running! 🚀"）

**Q: Bot 沒有回覆？**  
A: 檢查 Render 的 Logs，確認環境變數是否正確設定

**Q: Render 免費版會睡眠？**  
A: 免費版 15 分鐘無流量會進入休眠，第一則訊息可能需等 30 秒。升級付費版或使用 UptimeRobot 定時 ping 可解決

## 機器人功能說明

| 輸入 | 回覆內容 |
|------|----------|
| 中文句子 | 英文翻譯 + 3～5 個關鍵單字/片語（含詞性、中文意思、例句） |
| 英文句子 | 繁體中文翻譯 + 語法檢查 + 2～3 個相似句型 |
